# 反思评估组 - 快速启动指南

## 环境准备

### 1. 安装Python依赖
```bash
pip install -r requirements.txt
```

### 2. 配置DeepSeek API密钥
```bash
# Linux/Mac
export DEEPSEEK_API_KEY="your_api_key_here"

# Windows
set DEEPSEEK_API_KEY=your_api_key_here
```

### 3. 验证数据库连接
确保能访问PostgreSQL数据库：
- 主机: 10.13.1.26:5432
- 用户名: admin
- 密码: ABCabc@123

## 快速开始

### 方案1: 使用测试数据（推荐新手）

```bash
# 步骤1: 生成测试数据
python tests/generate_test_data.py --num-papers 3

# 步骤2: 运行主程序
python run.py

# 步骤3: 选择方案2（从文件读取）
# 输入: 2
# 输入目录: prompts

# 步骤4: 查看结果
# 结果保存在 prompts/result_test_paper_001.json 等文件中
```

### 方案2: 从数据库读取

```bash
# 运行主程序
python run.py --mode database

# 或处理指定论文
python run.py --mode database --paper-id paper_001
```

## 常见问题

### Q1: 如何生成测试数据？
```bash
python tests/generate_test_data.py --num-papers 5
```

### Q2: 如何指定输出目录？
```bash
python run.py --mode file --prompts-dir my_data
```

### Q3: 数据库连接失败怎么办？
检查：
1. 网络是否能访问10.13.1.26
2. 数据库名称是否正确（默认: paper_review）
3. 修改src/db/database.py中的DB_CONFIG

### Q4: DeepSeek API调用失败？
检查：
1. 环境变量DEEPSEEK_API_KEY是否设置
2. API密钥是否有效
3. 网络是否能访问api.deepseek.com

## 输入数据格式

每篇论文需要5个审计组的JSON文件（group_id: 2-6）：

```json
{
  "group_id": 2,
  "paper_id": "paper_001",
  "audit_results": [
    {
      "id": "item-001",
      "point": "审核点名称",
      "score": 85,
      "level": "Warning",
      "description": "问题描述",
      "evidence_quote": "原文引用",
      "location": {"section": "2.1", "line_start": 45},
      "suggestion": "改进建议"
    }
  ]
}
```

## 输出结果格式

```json
{
  "paper_id": "paper_001",
  "final_score": 82.5,
  "verdict": "论文质量良好，建议小修后录用",
  "critical_issues": [],
  "major_issues": [...],
  "minor_issues": [...],
  "needs_human_review": false
}
```
