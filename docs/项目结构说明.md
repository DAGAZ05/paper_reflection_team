# 反思评估组 - 重构后项目结构

## 项目概述

本项目是反思评估组的完整实现，采用模块化设计，支持从数据库或文件读取审计结果，进行冲突裁决、证据验证、优先级排序和导师对话生成。

## 核心功能模块

### 1. 统一数据库模块 (src/db/)
- **文件**: `database.py`
- **功能**:
  - 连接PostgreSQL数据库 (10.13.1.26)
  - 从agent_audits表读取审计结果
  - 获取论文内容用于证据验证
  - 保存反思评估结果
- **配置**:
  - 主机: 10.13.1.26
  - 用户名: admin
  - 密码: ABCabc@123

### 2. 统一API模块 (src/api/)
- **文件**: `deepseek_client.py`
- **功能**:
  - 统一使用DeepSeek API进行LLM调用
  - 支持冲突裁决和对话生成
  - 自动处理API请求和错误

### 3. 冲突裁决模块 (src/conflict_resolution/)
- **负责人**: 成员A
- **功能**:
  - 检测Agent评审结果之间的冲突
  - 使用LLM进行冲突裁决
  - 计算加权平均分
  - 生成Markdown评审报告
  - 证据真实性验证

### 4. 重复过滤模块 (src/deduplication/)
- **负责人**: 成员B
- **功能**:
  - 使用DBSCAN聚类进行重复过滤
  - 识别相似的审计意见
  - 去除冗余信息

### 5. 证据验证模块 (src/evidence_validation/)
- **负责人**: 成员B
- **功能**:
  - 精确匹配验证证据引用
  - 语义匹配验证（使用Sentence-Transformers）
  - 幻觉过滤：检测虚假证据

### 6. 导师对话生成模块 (src/dialogue_generation/)
- **负责人**: 成员C
- **功能**:
  - 根据领域和问题严重性构建导师人设
  - 生成温和专业的指导性对话
  - 对话质量自动评分

### 7. 优先级排序模块 (src/priority_sorting/)
- **负责人**: 成员D
- **功能**:
  - 基于规则引擎的优先级排序
  - 检测置信度低、证据缺失等触发条件
  - 检测多组Agent冲突
  - 生成人工复核标记

## 主运行程序 (run.py)

### 支持两种运行方案

#### 方案1: 从数据库读取
```bash
# 交互式选择
python run.py

# 命令行指定
python run.py --mode database

# 处理指定论文
python run.py --mode database --paper-id paper_001
```

从PostgreSQL数据库的`agent_audits`表读取数据：
- 表结构: `(paper_id, group_id, result_json, created_at)`
- 同一paper_id的5个group_id (2-6) 对应5个审计组
- result_json格式见docs/work_week2.txt

#### 方案2: 从文件读取
```bash
# 交互式选择
python run.py

# 命令行指定
python run.py --mode file --prompts-dir prompts
```

从prompts文件夹读取JSON文件：
- 文件命名: `{paper_id}_group_{group_id}.json`
- 每篇论文需要5个JSON文件（group_id: 2-6）

### 数据流程

```
审计结果输入 (数据库/文件)
    ↓
[成员D] 优先级排序 + 复核标记
    ↓
[成员A] 冲突裁决 + 整体评分
    ↓
[成员B] 证据验证 + 去重过滤
    ↓
[成员C] 导师对话生成
    ↓
最终评审报告 (保存到数据库/文件)
```

## 技术栈

- **框架**: FastAPI, Pydantic
- **LLM**: DeepSeek API
- **NLP**: Sentence-Transformers, scikit-learn
- **数据库**: asyncpg (PostgreSQL)
- **HTTP**: httpx
- **测试**: pytest

## 错误处理

项目包含完善的错误处理机制：

1. **数据库连接错误**: 自动重试和日志记录
2. **API调用失败**: 捕获HTTP错误并记录详细信息
3. **数据格式错误**: 验证输入数据格式，跳过无效数据
4. **模块初始化失败**: 记录错误并优雅退出
5. **文件读写错误**: 捕获IO异常并提示用户

## 注意事项

1. **API密钥**: 必须配置DEEPSEEK_API_KEY环境变量
2. **数据库连接**: 确保能访问10.13.1.26:5432
3. **数据格式**: 输入JSON必须符合work_week2.txt中的格式
4. **审计组数量**: 每篇论文必须有5个审计组的结果（group_id: 2-6）
5. **证据引用**: 所有Warning/Critical级别的问题必须包含evidence_quote
