# 反思评估组代码合并说明

## 合并概述

本次合并工作按照《反思评估组四人精细化周计划表》的分工，将四位成员的独立工作整合到项目根目录，形成统一的代码库。

## 合并完成的工作

### 1. 代码模块合并 (src/)

#### 成员A的模块
- `conflict_resolver.py` - 冲突裁决核心模块
- `llm_client.py` - LLM客户端
- `database_a.py` - 数据库客户端（重命名避免冲突）

#### 成员B的模块
- `evidence.py` - 证据验证模块
- `dedup.py` - 重复过滤模块
- `config_b.py` - 配置文件（重命名）
- `utils_b.py` - 工具函数（重命名）
- `database_b.py` - 数据库模块（重命名）

#### 成员C的模块
- `dialogue_engine.py` - 导师对话生成引擎
- `config_c.py` - 配置文件（重命名）
- `database_c.py` - 数据库模块（重命名）

#### 成员D的模块
- `review_engine.py` - 优先级排序和复核标记引擎

### 2. 统一数据模型 (src/models.py)

创建了统一的数据模型文件，整合了所有成员的数据结构：

- **枚举类型**: AuditLevel, ConflictType, AgentType
- **成员B的模型**: Section, EvidenceItem, EvidenceSpan, DedupItem, DedupCluster
- **成员A的模型**: AgentResultData, AgentResult, ConflictResolutionRequest, ConflictResolutionResponse
- **成员D的模型**: AuditResult, ReviewMarkResult, SortedAuditResult
- **成员C的模型**: PrioritizedIssue, MentorDialogue, ReflectionResult

### 3. 主编排器 (src/main.py)

创建了`ReflectionJudgeOrchestrator`类，协调四个模块的工作流程：

```python
数据流程：
Agent评审结果
  ↓
[成员D] 优先级排序 + 复核标记
  ↓
[成员A] 冲突裁决 + 整体评分
  ↓
[成员C] 导师对话生成
  ↓
最终评审报告
```

### 4. 测试文件合并 (tests/)

- `test_conflict_resolver.py` - 成员A的测试
- `test_review_engine.py` - 成员D的测试
- `conftest_a.py` - 测试配置

### 5. Prompts模板 (prompts/)

- `conflict_resolution/` - 成员A的冲突裁决提示词模板

### 6. 依赖管理 (requirements.txt)

整合了所有成员的依赖包：
- FastAPI, Pydantic (Web框架)
- LiteLLM (LLM调用)
- Sentence-Transformers, scikit-learn (NLP)
- asyncpg (数据库)
- pytest (测试)

## 接口兼容性处理

### 命名冲突解决

由于多个成员都有config.py、database.py等同名文件，采用了重命名策略：

- `config.py` → `config_b.py`, `config_c.py`
- `database.py` → `database_a.py`, `database_b.py`, `database_c.py`
- `utils.py` → `utils_b.py`

### 导入路径更新

更新了所有模块的导入语句，使其引用统一的models.py和重命名后的配置文件：

```python
# 原来
from .config import EvidenceConfig
from .models import EvidenceItem

# 更新后
from .config_b import EvidenceConfig
from .models import EvidenceItem
```

## 模块接口说明

### ConflictResolver (成员A)

```python
async def resolve_conflicts(request: ConflictResolutionRequest) -> ConflictResolutionResponse
```

输入：Agent评审结果列表
输出：冲突裁决结果、整体评分、Markdown报告

### Deduplicator (成员B)

```python
def dedup(paper_id: str, items: List[DedupItem]) -> DedupResult
```

输入：待去重的审计条目
输出：聚类结果和噪声点

### EvidenceValidator (成员B)

```python
def validate_item(item: EvidenceItem, sections: List[Section]) -> EvidenceValidationResult
```

输入：证据条目和论文章节
输出：验证结果（精确匹配或语义匹配）

### DialogueEngine (成员C)

```python
async def generate_dialogue(field: str, issues: List[PrioritizedIssue]) -> MentorDialogue
```

输入：论文领域和优先级问题列表
输出：导师对话内容

### ReviewDecisionEngine (成员D)

```python
def process_audit_results(raw_audit_list: List[Dict]) -> Tuple[List[SortedAuditResult], List[ReviewMarkResult]]
```

输入：原始审计结果列表
输出：排序后的结果和复核标记

## 使用示例

```python
from src import ReflectionJudgeOrchestrator

# 创建编排器
orchestrator = ReflectionJudgeOrchestrator()

# 处理论文评审
result = await orchestrator.process_paper_review(
    paper_id="paper_001",
    paper_title="基于深度学习的软件缺陷预测研究",
    paper_content="论文内容...",
    agent_results=[
        {
            "agent_info": {"name": "逻辑审计组", "version": "v1.0"},
            "result": {
                "score": 85,
                "audit_level": "Info",
                "comment": "论文逻辑清晰",
                "suggestion": "建议补充更多实验数据"
            }
        }
    ],
    field="软件工程"
)

# 查看结果
print(f"最终得分: {result.final_score}")
print(f"评审结论: {result.verdict}")
print(f"是否需要人工复核: {result.needs_human_review}")
```
