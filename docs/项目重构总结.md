## 数据流程

```
输入: agent_audits表 / JSON文件
  ↓
读取5个审计组的结果 (group_id: 2-6)
  ↓
[成员D] 优先级排序 + 复核标记
  - 计算排序得分
  - 检测复核触发条件
  - 生成ReviewMarkResult
  ↓
[成员A] 冲突裁决 + 整体评分
  - 检测冲突
  - LLM裁决
  - 计算加权平均分
  - 证据验证
  ↓
[成员B] 去重 + 证据验证
  - DBSCAN聚类去重
  - 精确/语义匹配
  - 幻觉过滤
  ↓
[成员C] 导师对话生成
  - 提取优先级问题
  - 构建导师人设
  - 生成对话
  ↓
输出: reflection_results表 / result_{paper_id}.json
```

## 使用示例

### 示例1: 生成测试数据并运行
```bash
# 1. 生成3篇论文的测试数据
python tests/generate_test_data.py --num-papers 3

# 2. 运行程序（交互式）
python run.py
# 选择: 2 (从文件读取)
# 输入目录: prompts

# 3. 查看结果
ls prompts/result_*.json
```

### 示例2: 从数据库读取
```bash
# 处理所有论文
python run.py --mode database

# 处理指定论文
python run.py --mode database --paper-id paper_001
```

## 错误处理

项目包含完善的错误处理：
1. 数据库连接失败 → 记录错误并退出
2. API调用失败 → 捕获HTTP错误并记录
3. 数据格式错误 → 跳过无效数据并继续
4. 模块初始化失败 → 记录错误并退出
5. 文件读写错误 → 捕获IO异常并提示
