# 反思评估组 - 最终使用说明

## ✅ 项目已完成并可用

所有功能已实现，代码已清理，可以直接投入使用。

## 快速开始（3步）

### 步骤1: 生成测试数据
```bash
python tests/generate_test_data.py --num-papers 3
```

**输出**:
- 在`prompts/`文件夹生成15个JSON文件（3篇论文 × 5个审计组）
- 文件命名: `test_paper_001_group_2.json` 到 `test_paper_003_group_6.json`

### 步骤2: 运行评审程序
```bash
python run.py
```

**交互式选择**:
```
请选择运行方案:
1. 从PostgreSQL数据库读取 (agent_audits表)
2. 从prompts文件夹读取JSON文件
0. 退出

请输入选项 (0-2): 2
请输入JSON文件目录 (默认: prompts): [直接回车]
```

**或使用命令行参数**:
```bash
# 从文件读取
python run.py --mode file --prompts-dir prompts

# 从数据库读取
python run.py --mode database

# 处理指定论文
python run.py --mode database --paper-id paper_001
```

### 步骤3: 查看评审报告
```bash
# 查看生成的Markdown报告
ls reports/
# 输出: review_report_test_paper_001_20260305_143022.md

# 查看JSON结果
ls prompts/result_*.json
# 输出: result_test_paper_001.json
```

### 表结构

#### agent_audits表
```sql
CREATE TABLE agent_audits (
    paper_id VARCHAR(255),
    group_id INT,  -- 2-6 (格式、逻辑、代码、实验、文献)
    result_json JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);
```

#### paper_sections表（用于证据验证）
```sql
CREATE TABLE paper_sections (
    section_id VARCHAR(255),
    paper_id VARCHAR(255),
    section_name VARCHAR(255),
    section_content TEXT,
    content_vector VECTOR  -- 可选，用于语义搜索
);
```

#### reflection_results表
```sql
CREATE TABLE reflection_results (
    paper_id VARCHAR(255) PRIMARY KEY,
    final_score FLOAT,
    verdict TEXT,
    result_json JSONB,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

## 常见问题

### Q1: 测试数据生成失败？
```bash
# 确保在项目根目录
cd project/
python tests/generate_test_data.py --num-papers 3
```

### Q2: 运行时找不到模块？
```bash
# 确保在项目根目录运行
cd project/
python run.py
```

### Q3: DeepSeek API调用失败？
```bash
# 检查环境变量
echo $DEEPSEEK_API_KEY  # Linux/Mac
echo %DEEPSEEK_API_KEY%  # Windows

# 如果未设置，设置环境变量
export DEEPSEEK_API_KEY="your_key"
```

### Q4: 数据库连接失败？
- 检查网络是否能访问 10.13.1.26
- 确认数据库名称是否正确
- 如需修改，编辑 `src/db/database.py`

### Q5: 没有生成Markdown报告？
- 检查 `reports/` 文件夹是否存在
- 查看程序日志中的错误信息
- 确认有写入权限
